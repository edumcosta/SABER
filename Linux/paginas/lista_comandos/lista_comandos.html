Redirecionamentos

Esta seção explica o funcionamento dos recursos de direcionamento de entrada e saída do sistema GNU/Linux.

> Caracter '>'

Redireciona a saída padrão de um programa/comando/script para algum dispositivo ou arquivo ao invés do dispositivo de saída padrão (tela). Quando é usado com arquivos, este redirecionamento cria ou substitui o conteúdo do arquivo.

Por exemplo, você pode usar o comando ls para listar arquivos e usar ls >listagem para enviar a saída do comando para o arquivo listagem. Use o comando cat para visualizar o conteúdo do arquivo listagem.

> Caracter ">>"

Redireciona a saída padrão de um programa/comando/script para algum dispositivo ou adiciona as linhas ao final de arquivo ao invés do dispositivo de saída padrão (tela). A diferença entre este redirecionamento duplo e o simples, é se caso for usado com arquivos, adiciona a saída do comando ao final do arquivo existente ao invés de substituir seu conteúdo.

Por exemplo, você pode acrescentar a saída do comando ls ao arquivo listagem do capítulo anterior usando ls / >>listagem. Use o comando cat para visualizar o conteúdo do arquivo listagem.

> Caracter '<'

Direciona a entrada padrão de arquivo/dispositivo para um comando. Este comando faz o contrário do anterior, ele envia dados ao comando.

Você pode usar o comando cat < teste.txt para enviar o conteúdo do arquivo teste.txt ao comando cat que mostrará seu conteúdo ou por exemplo, mandar o conteudo de um arquivo para o comando "mail", $ mail uira@certificacaolinux.com.br < e-mail.txt

.

> Caracter "|"

Redireciona a saída padrão de um programa/comando/script para algum dispositivo ou adiciona as linhas ao final de arquivo ao invés do dispositivo de saída padrão (tela). A diferença entre este redirecionamento duplo e o simples, é se caso for usado com arquivos, adiciona a saída do comando ao final do arquivo existente ao invés de substituir seu conteúdo.

Por exemplo, você pode acrescentar a saída do comando ls ao arquivo listagem do capítulo anterior usando ls / >>listagem. Use o comando cat para visualizar o conteúdo do arquivo listagem.

Coringas

Coringas (ou referência global) é um recurso usado para especificar um ou mais arquivos ou diretórios do sistema de uma só vez. Este é um recurso permite que você faça a filtragem do que será listado, copiado, apagado, etc. São usados 4 tipos de coringas no GNU/Linux:

Coringa Resultado
* Faz referência a um nome completo/restante de um arquivo/diretório.
? Faz referência a uma letra naquela posição.
[padrão] Faz referência a uma faixa de caracteres de um arquivo/diretório. A procura pelos caracteres diferencia maisculos de minúsculas [a-z A-Z]. Os padrões podem ser:
[a-z][0-9] Faz referência a caracteres de a até z seguido de um caracter de 0 até 9.
[a,z][1,0] Faz a referência aos caracteres a e z seguido de um caracter 1 ou 0 naquela posição.
[a-z,1,0] Faz referência a intervalo de caracteres de a até z ou 1 ou 0 naquela posição.
^ Faz referência a uma letra naquela posição. Por exemplo [^abc] faz referência a qualquer caracter exceto a, b e c.
{padrões} Expande e gera strings para pesquisa de padrões de um arquivo/diretório.
X{ab,01} Faz referência a seqüencia de caracteres Xab ou X01.
X{a-z,10} Faz referencia a seqüencia de caracteres Xa-z e X10.

Exemplo de coringas:

O diretório /usr/teste: teste1.txt, teste2.txt, teste3.txt, teste4.new, teste5.new.

ls *.txt Retorna todos os arquivos onde o final seja igual ".txt".
ls teste?.txt Retorna todos os arquivos que comecem com "teste" e na sexta posição, possua qualquer caracter e após esse caracter, o final seja igual ".txt"
ls teste[1-3].txt Retorna todos os arquivos que comecem com "teste" e na sexta posição possua um dos caracteres, 1 ou 2 ou 3 e após esse caracter, o final seja igual ".txt".
ls teste[1-3].txt Retorna todos os arquivos que comecem com "teste" e na sexta posição possua um dos caracteres, 1 ou 2 ou 3 e após esse caracter, o final seja igual ".txt".
ls teste[4,5].* Retorna todos os arquivos que comecem com "teste" e na sexta posição possua um dos caracteres, 4 ou 5 e depois desse caracter seja qualquer valor.

> ls

  • diretórios ('/')
  • soquete ('=')

    > rm

    Apaga arquivos. Também pode ser usado para apagar diretórios e sub-diretórios vazios ou que contenham arquivos.

    rm [opções][caminho][arquivo/diretório] [caminho1][arquivo1/diretório1]

    Opções Resultado
    diretório/arquivo Arquivo que será apagado.
    -i, --interactive Pergunta antes de remover, esta é ativada por padrão.
    -v, --verbose Mostra os arquivos na medida que são removidos.
    -r, --recursive Usado para remover arquivos em sub-diretórios. Esta opção também pode ser usada para remover sub-diretórios.
    -f, --force Remove os arquivos sem perguntar.
    -- arquivo Remove arquivos/diretórios que contém caracteres especiais. O separador "--" funciona com todos os comandos do shell e permite que os caracteres especiais como "*", "?", "-", etc. sejam interpretados como caracteres comuns.

    Abaixo o exemplos do comando rm.

    Exemplo Resultado
    rm teste.txt Apaga o arquivo teste.txt no diretório atual.
    rm *.txt Apaga todos os arquivos do diretório atual que terminam com .txt.
    rm *.txt teste.novo Apaga todos os arquivos do diretório atual que terminam com .txt e também o arquivo teste.novo.
    rm -rf /tmp/teste/* Apaga todos os arquivos e sub-diretórios do diretório /tmp/teste mas mantém o sub-diretório /tmp/teste.
    rm -rf /tmp/teste paga todos os arquivos e sub-diretórios do diretório /tmp/teste, inclusive /tmp/teste.
    rm -f -- teste Remove o arquivo de nome teste.

    > cp

    Copia arquivos e diretórios

    cp [opções] [origem] [destino]

    Opções Resultado
    -i, --interactive Pergunta antes de substituir um arquivo existente.
    -f, --force Não pergunta, substitui todos os arquivos caso já exista.
    -r Copia arquivos dos diretórios e subdiretórios da origem para o destino. É recomendável usar -R ao invés de -r.
    -R, --recursive Copia arquivos e sub-diretórios (como a opção -r) e também os arquivos especiais FIFO e dispositivos.
    -v, --verbose Mostra os arquivos enquanto estão sendo copiados.
    -s, --simbolic-link Cria link simbólico ao invés de copiar.
    -l, --link Faz o link no destino ao invés de copiar os arquivos.
    -p, --preserve Preserva atributos do arquivo, se for possível.
    -u, --update Copia somente se o arquivo de origem é mais novo que o arquivo de destino ou quando o arquivo de destino não existe.

    Abaixo o exemplos do comando cp.

    Exemplo Resultado
    cp teste.txt teste1.txt Copia o arquivo teste.txt para teste1.txt.
    cp teste.txt /tmp Copia o arquivo teste.txt para dentro do diretório /tmp.
    cp * /tmp Copia todos os arquivos do diretório atual para /tmp.
    cp /bin/* Copia todos os arquivos do diretório /bin para o diretório em que nos encontramos no momento.
    cp -R /bin /tmp Copia o diretório /bin e todos os arquivos/sub-diretórios existentes para o diretório /tmp.
    cp -R /bin/* /tmp Copia todos os arquivos do diretório /bin (exceto o diretório /bin) e todos os arquivos/sub-diretórios existentes dentro dele para /tmp.
    cp -R /bin /tmp Copia todos os arquivos e o diretório /bin para /tmp.

    > mv

    Move ou renomeia arquivos e diretórios. O processo é semelhante ao do comando cp mas o arquivo de origem é apagado após o término da cópia.

    mv [opções] [origem] [destino]

    Opções Resultado
    -f, --force Substitui o arquivo de destino sem perguntar.
    -i, --interactive Pergunta antes de substituir. É o padrão.
    -v, --verbose Mostra os arquivos que estão sendo movidos.
    -u, --update Move somente arquivos antigos, ou novos arquivos.

    Abaixo o exemplos do comando mv.

    Exemplo Resultado
    mv teste.txt teste1.txt Muda o nome do arquivo teste.txt para teste1.txt.
    mv teste.txt /tmp Move o arquivo teste.txt para /tmp. Lembre-se que o arquivo de origem é apagado após ser movido.
    mv teste.txt teste.new Copia o arquivo teste.txt por cima de teste.new (supondo que teste.new já exista) e apaga teste.txt após terminar a cópia.

    Manipulação Diretórios e Arquivos

    > ln

    Cria links para arquivos e diretórios no sistema. O link é um mecanismo que faz referência a outro arquivo ou diretório em outra localização.

    ln [opções] [origem] [link]

    Opções Resultado
    origem Diretório ou arquivo de onde será feito o link.
    link Nome do link que será criado.
    -s Cria um link simbólico. Usado para criar ligações com o arquivo/diretório de destino.
    -v Mostra o nome de cada arquivo antes de fazer o link.
    -d Cria um hard link para diretórios. Somente o root pode usar esta opção.

    Existem 2 tipos de links: simbólicos e hardlinks.

    Link Simbólico O cria um arquivo especial no disco (do tipo link) que tem como conteúdo o caminho para chegar até o arquivo alvo (isto pode ser verificado pelo tamanho do arquivo do link). Use a opção -s para criar links simbólicos.
    hardlink O hardlink faz referência ao mesmo inodo do arquivo original, desta forma ele será perfeitamente idêntico, inclusive nas permissões de acesso, ao arquivo original.

    Ao contrário dos links simbólicos, não é possível fazer um hardlink para um diretório ou fazer referência a arquivos que estejam em partições diferentes.

    Exemplos do comando "ln".

    Exemplo Resultado

    Compressão e Descompressão de Arquivos

    > gzip

    Este comando compacta/descompacta arquivos.

    gzip [opções] [arquivos]

    Opções Resultado
    -c Grava o resultado na saída padrão e mantém o arquivo original inalterado.
    -d Descompacta (igual ao comando gunzip).
    -l Lista informações sobre os arquivos.
    -r compacta/descompacta recursivamente (navega a estrutura de diretórios recursivamente).
    -t Verifica a integridade do arquivo compactado.
    -número (1 a 9, o padrão é 6) Define o nível de compressão (1 menor, 9 maior).
    -v Mostra a taxa de compressão do arquivo.

    Abaixo o exemplos do comando gzip.

    Exemplo Resultado
    gzip teste1.txt Compacta o arquivo teste1.txt e gera o arquivo teste1.txt.gz. O arquivo original e apagado.
    gzip -c teste1.txt Compacta o arquivo teste1.txt, gera o arquivo teste1.txt.gz e o arquivo teste1.txt permanece presente no diretório.
    gzip -1 filename Compacta o arquivo com nível de compressão = 1.
    gzip filename1 filename2 Compacta os arquivos informados.
    gzip -r Pasta1 Compacta o diretório "Pasta1" de forma recursiva.
    gzip -rcv Pasta1 Compacta o diretório "Pasta1" de forma recursiva e mostra a taxa de compressão.
    gzip -rcv Pasta1 >Pasta1_Compactada.gz Compacta o diretório "Pasta1" de forma recursiva, mostra a taxa de compressão e cria o arquivo Pasta1_Compactada.gz.
    gzip -d filename.gz Descompacta o arquivo.

    > scp

    SCP (Secure Copy) é um protocolo de rede para transferências de arquivos.

    scp [other options] [source username@IP]:/[directory and file name] [destination username@IP]:/[destination directory]

    Opções Resultado
    [other options] São modificadores que você pode adicionar ao comando SCP. Vamos mostrar os mais populares dele mais abaixo.
    [source username@IP] É o nome do usuário e o IP da máquina que tem o arquivo que você quer. Seria algo como root@123.123.123.12.
    :/ Informa o comando SCP que você está prestes a digitar no diretório fonte.
    [directory and file name] É onde o arquivo está localizado e o nome dele. Se parece com algo como /users/Edward/Desktop/SCP.png
    [destination username@IP] É o nome do usuário e do IP da máquina de destino.
    [destination directory] É o diretório onde o arquivo será salvo.

    Abaixo exemplo do comando:

    scp -p root@162.168.1.1:/media/scp.png edward@162.168.1.2:/desktop/destination

    scp CAMINHO\ARQUIVO [Windows] edward@162.168.1.2:/desktop/destination

    scp CAMINHO/ARQUIVO [Linux] edward@162.168.1.2:/desktop/destination

    As outras opções do comando scp.

    Opções Resultado
    -P Possibilita especificar uma entrada diferente para o servidor (a porta padrão para a porta TCP para o comando é a 22).
    -c Permite especificar a encriptação do algoritmo que o cliente vai usar. Alguns dos valores que você pode usar são ‘aes256-ctr’, ‘aes256-cbc’, ‘blowfish-cbc’, ‘arcfour’, ‘arcfour128’, ‘arcfour256’, ‘cast128-cbc’, aes128-ctr’, ‘aes128-cbc’, ‘aes192-ctr’, ‘aes192-cbc’ e 3des-cbc’. A opção padrão na configuração do shell é ‘AnyStdCipher’.
    -q Executa a operação em modo silencioso, o que significa que apenas os erros críticos é que serão mostrados.
    -r Cópia recursiva, que vai incluir todos os subdiretórios.
    -p Preserva os tempos iniciais de modificação e atributos do arquivo. .
    -u Apaga a fonte do arquivo logo depois que a transferência for completada.
    -c Habilita a compressão de dados enquanto a operação de transferência está sendo executada.

    > find

    O comando find no UNIX é um utilitário de linha de comando para percorrer uma hierarquia de arquivos.
    Ele pode ser usado para localizar arquivos e diretórios e executar operações subsequentes neles.
    Ele suporta pesquisa por arquivo, pasta, nome, data de criação, data de modificação, proprietário e permissões. Ao usar o '-exec', outros comandos UNIX podem ser executados em arquivos ou pastas encontrados.

    $ find [onde começar a pesquisar] [expressão determina o que encontrar] [-opções] [o que encontrar]

    Opções Resultado
    -L Redireciona para a existência de URL em HTTPS, caso não informado na linha de comando.
    Exemplo: curl -L www.xxxxx.com.br/
    -i Apresenta mais informações de resultado.
    Exemplo: curl -L -i www.xxxxx.com.br/
    -I Apresenta somente as informações de cabeçalho.
    Exemplo: curl -L -I www.xxxxx.com.br/
    -O Realiza download de um arquivo e grava com o mesmo nome.
    Exemplo: curl -O https://releases.ubuntu.com/20.04.1/ubuntu-20.04.1-desktop-amd64.iso ubuntu20.iso
    -s Omite as informações de progresso.
    Exemplo: curl -O -s https://releases.ubuntu.com/20.04.1/ubuntu-20.04.1-desktop-amd64.iso ubuntu20.iso
    -C Continua o download caso seja interrompido.
    Exemplo: curl -C - -O https://releases.ubuntu.com/20.04.1/ubuntu-20.04.1-desktop-amd64.iso ubuntu20.iso
    Observação: o "-" após a opção -C indica que o download deve iniciar do ponto onde parou.
    -X [METODO HTTP] Executa um método HTTP, como por exemplo realizar o upload de uma arquivo.
    Exemplo: curl -C - -O curl -X POST -F "arquivo=@caminho/local/arquivo" URL_do_servidor
    curl -X POST -F "arquivo=@/caminho/local/meu_arquivo.txt" http://exemplo.com/upload.php.
    Opções Resultado
    -exec CMD O arquivo sendo pesquisado que atende aos critérios acima e retorna 0 como seu status de saída para execução bem-sucedida do comando.
    -ok CMD Funciona da mesma forma que -exec, exceto que o usuário é solicitado primeiro.
    -inum N Procura por arquivos com número de inode [N corresponde ao inode].
    -links N Pesquisa arquivos com links [N corresponde ao link].
    -name demo Pesquise por arquivos que são especificados por "demo".
    -iname Ignora diferença de maiúsculas e miniúsculas.
    -newer file Pesquise por arquivos que foram modificados/criados do arquivo "file".
    -perm octal Procure arquivos com permissão formato "octal".
    -user usuário Procure arquivos pertencentes ao nome de usuário ou ID 'nome'.
    -print Exibe o nome do caminho dos arquivos encontrados usando o restante dos critérios.
    -empty Pesquisa por arquivos e diretórios vazios.
    -type Procura por um tipo de arquivo:
    f: a regular file
    d: directory
    l: symbolic link
    c: character devices
    b: block devices
    p: named pipe (FIFO)
    s: socket

    Exemplos:

    • find /home -name tecmint.txt
    • find /home -iname tecmint.txt
    • find / -type d -name Tecmint
    • find . -type f -perm 0777 -print
    • find / -type f ! -perm 777

    > Rsync

    Pagina de documentação do "rsync" :

    Para instalar o "rsync": "sudo apt install rsync"
    Para instalar o "grsync", versão GUI : "sudo apt-get install grsync"

    Para iniciar o rsync como um serviço, crie o arquivo, vazio, "/etc/rsyncd.conf" e copie "/lib/systemd/system/rsync.service" para "/etc/systemd/system/rsync.service".
    Reiniciar o serviço: "sudo systemctl restart rsync".

    Principais opções "rsync":

    Opção Descrição
    -v Mostra a saída da execução do comando.
    -h Mostra a saída da execução do comando, formato humano.
    -d Cópia de diretórios de forma não recursiva.
    -u Pula os arquivos que são mais recentes no destino.
    -r Copia arquivos recursivamente, mas sem preservar as permissões e links.
    -a Copia arquivos recursivamente e preserva links simbólicos, permissões de arquivos, propriedades de usuário e grupo (ownership) e timestamps.
    -p Preservar as permissões.
    -E Preserva a propriedade de execução.
    -z Aplica processo de compactação das informações.
    -e Especificar o shell remoto a ser usado (rsh, ssh).
    -c ou -–checksum Calcula os checksums dos arquivos para verificar se eles são iguais ao transferi-los. Utilizada para verificar a integridade dos dados copiados.
    -b ou -–backup Arquivos existentes no destino serão deletados conforme a transferência progride.
    -P ou -–progress Mostra como saída o progresso da execução.
    --include Permite especificar arquivos ou diretórios.
    --delete Exclui um arquivo ou diretório no destino caso ele não exista na origem.
    --exclude=PATTERN Exclui o arquivo que coincide com o PATTERN.
    -–remove-source-files Exclui os arquivos no diretório de origem após o término.
    -n / -–dry-run Operar em modo teste / simulação.
    -–log-file Local e arquivo de log.

    Temos ainda dentro desse comando as definições de:

    • [SOURCE] : – Caminho de "origem", arquivos, diretórios que pode ser o host local ou um host remoto.
    • [DESTINATION] – Caminho de "destino" onde os arquivos serão copiados e que pode ser o host local ou um host remoto.
